# Goal Templates - OKR Methodiek

## Introductie

Dit document beschrijft de **OKR (Objectives and Key Results)** structuur voor goal setting in het Falah platform. OKR is een bewezen methodiek die gebruikt wordt door bedrijven en individuen voor effectieve doelstellingen.

## OKR Methodiek

### Wat is OKR?

**OKR = Objectives and Key Results**

- **Objective**: Een kwalitatieve, inspirerende doelstelling (wat wil je bereiken?)
- **Key Result**: Meetbare resultaten die aangeven of je het Objective hebt bereikt (hoe meet je succes?)
- **Initiative**: Concrete acties/taken die je onderneemt om de Key Results te bereiken

### Waarom OKR?

1. **Bewezen methodiek**: Gebruikt door Google, Intel, en vele andere succesvolle organisaties
2. **Natuurlijker**: Objectives zijn inspirerend, Key Results zijn meetbaar
3. **Flexibeler**: Minder kunstmatige lagen dan de vorige structuur
4. **Duidelijker**: Directe link tussen doel en actie
5. **Meetbaar**: Key Results zijn kwantificeerbaar en trackbaar

## Structuur

```
Life Domain → Objective → Key Result → Initiative (Individual Goal)
```

### Conceptuele Betekenis

- **Life Domain**: Breed domein (Religie, Familie, Werk, etc.)
- **Objective**: Inspirerend, kwalitatief doel binnen een domain (bijv. "Dagelijks gebed consistent verrichten")
- **Key Result**: Meetbaar resultaat dat succes van het Objective aangeeft (bijv. "Fajr 30 dagen op tijd")
- **Initiative/Individual Goal**: Concrete actie/taak om een Key Result te bereiken (bijv. "Fajr alarm zetten om 5:30")

### Template vs. User-Specific

- **Templates (shared)**: Objectives en Key Results zijn templates - ze worden gedeeld door alle gebruikers
- **User-Specific**: UserObjectiveInstances en Initiatives zijn gebruiker-specifiek - elke gebruiker heeft zijn eigen instances en acties

## Top 5 Life Domains (Start)

We starten met de 5 meest relevante life domains voor de doelgroep:

1. **RELIGION** (Religie / ʿIbādah) - Fundamenteel voor Falah
2. **PERSONAL_GROWTH** (Persoonlijke Groei / Tazkiyyah) - Kern van het platform
3. **FAMILY** (Gezin / Familie) - Belangrijk in islamitische context
4. **HEALTH** (Gezondheid) - Essentieel voor functioneren en groeien
5. **FINANCE** (Financiën) - Financiële stabiliteit en verantwoordelijkheid

## Database Structuur

### Tables

```sql
-- Objectives (templates)
objectives (
  id BIGSERIAL PRIMARY KEY,
  life_domain_id BIGINT NOT NULL,
  title_nl VARCHAR(255) NOT NULL,
  title_en VARCHAR(255) NOT NULL,
  description_nl TEXT,
  description_en TEXT,
  order_index INTEGER NOT NULL,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
)

-- Key Results (templates)
key_results (
  id BIGSERIAL PRIMARY KEY,
  objective_id BIGINT NOT NULL,
  title_nl VARCHAR(255) NOT NULL,
  title_en VARCHAR(255) NOT NULL,
  description_nl TEXT,
  description_en TEXT,
  target_value DECIMAL(10,2),
  unit VARCHAR(50), -- 'dagen', 'uren', 'euro', 'percentage', etc.
  order_index INTEGER NOT NULL,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
)

-- User Objective Instances (user-specific)
user_objective_instances (
  id BIGSERIAL PRIMARY KEY,
  user_id BIGINT NOT NULL,
  objective_id BIGINT NOT NULL,
  started_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  completed_at TIMESTAMP,
  CONSTRAINT uk_user_objective UNIQUE (user_id, objective_id)
)

-- Initiatives/Goals (user-specific)
initiatives (
  id BIGSERIAL PRIMARY KEY,
  user_id BIGINT NOT NULL,
  key_result_id BIGINT NOT NULL,
  user_objective_instance_id BIGINT NOT NULL,
  title VARCHAR(255) NOT NULL,
  description TEXT,
  status VARCHAR(50) NOT NULL DEFAULT 'ACTIVE', -- ACTIVE, COMPLETED, ARCHIVED
  target_date DATE,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
)

-- Key Result Progress (user-specific)
key_result_progress (
  id BIGSERIAL PRIMARY KEY,
  user_id BIGINT NOT NULL,
  key_result_id BIGINT NOT NULL,
  user_objective_instance_id BIGINT NOT NULL,
  current_value DECIMAL(10,2),
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  CONSTRAINT uk_user_key_result UNIQUE (user_id, key_result_id, user_objective_instance_id)
)
```

## Implementatie Details

### Database Seeding

De seeding gebeurt via een Flyway migration die:
1. Voor elk life domain de objectives aanmaakt
2. Voor elke objective de key results aanmaakt
3. Voorbeelden van initiatives kunnen worden toegevoegd (optioneel, meestal user-specific)

### Position Values

- **Objectives**: `order_index` binnen een life domain (1, 2, 3, 4, 5)
- **Key Results**: `order_index` binnen een objective (1, 2, 3, ...)

### Taalondersteuning

Alle templates hebben zowel Nederlandse (`title_nl`, `description_nl`) als Engelse (`title_en`, `description_en`) titels en beschrijvingen.

### Tracking

- **Key Results**: Gebruikers kunnen hun progressie tracken via `key_result_progress` (current_value vs target_value)
- **Initiatives**: Gebruikers kunnen individuele acties/doelen stellen en volgen

---

## Gebruik

Wanneer een gebruiker op een life domain klikt in de Goal Navigator:
1. Ziet hij/zij de beschikbare Objectives
2. Klikt op een Objective → ziet Key Results met target values
3. Start een UserObjectiveInstance → kan progressie tracken per Key Result
4. Voegt Initiatives toe → concrete acties om Key Results te bereiken

De gebruiker wordt begeleid door deze OKR structuur, waarbij:
- Objectives inspirerend en kwalitatief zijn
- Key Results meetbaar en trackbaar zijn
- Initiatives concrete acties zijn die de gebruiker kan uitvoeren
